2. OBTAINING ALPACA API KEYS

Alpaca is the commission-free trading API brokerage that hosts our trading engine execution. To get your keys:

  1. Navigate to Alpaca's Signup Portal and create an account.
  2. Complete registration. On your main web portal dashboard, you will see a box on the right-hand panel labeled Your API Keys.
  3. By default, it is recommended to start with Paper Trading (simulated funds). Toggle the environment dropdown from "Live" to "Paper" to generate test credentials.
  4. Click Generate API Key (or View API Key). This will display:
    • API Key ID (public identifier, e.g., PK...)
    • Secret API Key (private token, visible only once upon generation)
  5. Copy and paste these strings into your API Settings panel inside our platform.
IMPORTANT SAFETY REMINDER Keep your Secret Key private at all times. Never share your secret keys or upload them to public source code files.

3. CORE PLATFORM FUNCTIONS

Here is an overview of the strategy settings and parameters available in your configuration console:

🛡️ Stop-Loss (SL) % Override The custom percentage below (for Buy orders) or above (for Sell orders) the entry price where the engine will trigger a risk-mitigating stop order. If set to 0, the stop-loss defaults to the opposite boundary of the 5-Minute Opening Range (standard ORB breakout).
🎯 Take-Profit (TP) % Override The custom target percentage where the engine will exit a profitable trade. If set to 0, the take-profit target will default to standard ORB strategy target calculations.
📊 Daily Trade Limit The maximum number of trades the bot is allowed to enter in a single trading day. Once this limit is reached, the bot suspends new entries and focuses on monitoring current active trades.
🧪 Dry-Run Mode A built-in simulator that mimics breakout triggers using live pricing, logging hypothetical entries, stops, and P&L results to your cloud history ledger without using any actual funds or placing orders with Alpaca. Perfect for testing before trading.
🔴 EOD Liquidation At 15:55:00 EST (5 minutes before NYSE market close), the system triggers an automatic liquidation event that cancels all pending breakout orders and closes all remaining open positions, protecting your capital from overnight holding risk.

4. THE ORB STRATEGY MECHANISM

The 15-Minute Opening Range Breakout (ORB) is a highly popular intraday strategy:

  • Calculation (09:30 AM - 09:45 AM EST): The engine tracks the stock's highest and lowest price during the first 15 minutes of NYSE trading. This forms the upper boundary (ORB High) and lower boundary (ORB Low).
  • Trigger Conditions:
    • Bullish Breakout (Buy): If the price breaks above the ORB High (after having been inside the range), a long position is initiated.
    • Bearish Breakout (Sell): If the price breaks below the ORB Low (after having been inside the range), a short position is initiated.
  • Bracket Protection: Upon entry, the system instantly submits a bracket order to Alpaca consisting of the entry order, an opposing stop-loss order (placed at the opposite range level or custom SL%), and a limit take-profit order.

5. SECURITY & DATA PRIVACY

We prioritize data privacy and user safety at every level of the platform:

  • API Key Encryption: Your Alpaca API Key and Secret are never stored as plain text. They are encrypted at rest using server-side AES-256 (Fernet) symmetric encryption before being written to Firestore.
  • No Client Access to Firestore: Access rules block any direct web reads/writes. The Firebase Admin SDK operates on the server-side, meaning only your authenticated user thread can access your secure profile information.
  • Account Deletion Option: Clicking "Delete Account" in Settings permanently purges all profile tokens, encrypted keys, settings, and historical trade logs from the Firebase database.
